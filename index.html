Stage

Remote System Engineer bij VanRoey.be

Stageopdracht

Vanroey.be beschikt over verschillende Azure tenants, hierdoor is het overzicht op vlak van financiën, rechten en resources in alle klantgerichte tenants en test omgevingen zoek. Als stageopdracht heb ik samen met een andere student de opdracht gekregen om deze Azure tenants te analyseren en een nieuwe tenant aan te maken waarop we alle subscriptions, resources en gebruikers van de eerder gemaakte tenants samenvoegen. Deze ‘master’ tenant richten we in volgens het Cloud Adoption Framework. Hierdoor is er een overzichtelijke structuur binnen de tenant waar in de toekomst ook verder op gebouwd kan worden.

Al snel werd de omvang van deze opdracht duidelijk en zijn we samen met onze stagebegeleider tot het besluit gekomen om ons enkel te focussen op het correct managen van de 'master' tenant. De eigenlijke migratie laten we over aan VanRoey zelf omdat er veel applicaties en dergelijke in productie omgevingen runnen. Ook ging het over zoveel maatwerk dat de kans te groot was dat we door onze, tot nu toe beperkte, kennis over Azure dingen verkeerd gingen doen.

Projecten

Infrastructure as Code Challenge

Voor het vak Emerging Technologies kregen we de opdracht om via een CI/CD pipeline automatisch een webserver naar Azure te deployen. Dit was een opdracht in groep (3 personen) maar mijn teamleden waren niet zo gekend in het concept 'IaC' dus heb ik ongeveer 75% van het werk gedaan.

Onze pipeline werkt via een jenkins server die verschillende .yaml files gebruikt om een webserver naar Azure te deployen. Deze files zijn terug te vinden via mijn github . Zoals u op de afbeelding kan zien stuurt de server een melding naar een discord server telkens de pipeline opgestart en ook een tweede melding met de status van de pipeline, succes of fail. Deze pipeline werd opgestart doormiddel van een web-hook zodat telkens wanneer we iets naar onze github pushen de pipeline automatisch start.

Smart Dispenser VanRoey.be

Voor het vak Project 4.0 hebben wij van VanRoey de opdracht gekregen voor een smart dispenser systeem te ontwerpen, dit idee kwam van Estée Lauder. Na een initiële analyse van VanRoey is er bepaald dat bedrijven dynamisch regels/beperkingen kunnen opleggen met betrekking tot de verdeling van producten (handgels, mondmaskers, ...). Ook zal dit systeem informatie bevatten met betrekking tot de verdeling van de producten en de voorraad van de dispensers, deze kan geraadpleegd worden met behulp van een webapplicatie.

Mijn aandeel in dit geheel is het opzetten van de Azure omgeving: vm's, netwerk, firewall, ... en het in elkaar steken van een pipeline in Azure DevOps. Deze pipeline build eerst de API en daarna deployed deze de applicatie naar onze Azure omgeving. Dit gebeurt telkens automatisch wanneer een aanpassing naar onze github gepushed word.

Meer informatie vindt u in onze documentatie. Deze bevat twee documenten: een plan van aanpak/analyse en een document dat onze realisaties beschrijft.

Project Hosting

De opdracht hier is het opstarten van een hosting platform voor web-applicaties in PHP en html5. Voor dit groepswerk werken we op een server die op de campus staat. Hierop gaan we een aantal virtuele machines instaleren, gebruik makend van VMware vSphere. We gaan dan gebruik maken van een docker swarm om zo verschillende klanten hun eigen omgeving te geven.

Mijn aandeel in het geheel is het mee aanmaken van virtuele machines, troubleshooting om geïnstalleerde software werkende te krijgen en het integreren van een ticketing systeem voor support voor onze klanten. Via deze link kan u het projectverslag terugvinden.
